# 🎥 Тестирование HLS потоков

## 🔧 Что было исправлено

### Основная проблема
HLS потоки (.m3u8) не поддерживаются нативно в веб-браузерах без специальных библиотек.

### Решение
1. ✅ Установлена библиотека `hls.js` для поддержки HLS в браузере
2. ✅ Создан специальный компонент `HLSVideoPlayer.web.tsx` для веб-платформы
3. ✅ Добавлена автоматическая детекция HLS потоков
4. ✅ Улучшенные сообщения об ошибках с отладочной информацией

## 📱 Как работает

### Автоматический выбор плеера
```typescript
// Для веб-платформы и HLS потоков (.m3u8)
{Platform.OS === 'web' && channel.stream_url.includes('.m3u8') ? (
  <HLSVideoPlayer />      // Использует hls.js
) : (
  <VideoPlayer />         // Стандартный Expo Video
)}
```

### Поддержка HLS
- **Веб**: Использует `hls.js` для декодирования HLS потоков
- **Мобильные**: Нативная поддержка HLS в iOS/Android
- **Fallback**: Информативные сообщения об ошибках

## 🌐 CORS и сетевые проблемы

### Возможные проблемы
1. **CORS** - сервер example.com может блокировать запросы
2. **HTTPS/HTTP** - смешанный контент в браузере
3. **Недоступность** - потоки могут быть offline

### Решения
1. Использовать HTTPS для всех потоков
2. Настроить CORS на сервере example.com
3. Использовать прокси для обхода CORS
