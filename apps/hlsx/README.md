# HLX Server

HLX Server - это Go-сервер для проксирования и модификации HLS потоков с улучшенной обработкой времени и длительностей сегментов.

## Новые параметры (сокращенные)

### Manifest endpoint: `/manifest`

- `s` - идентификатор потока (было `stream`)
- `dt` - удалить дату/время (было `delete_datetime`, `strip_pdt`)
- `rl` - удалить последний сегмент (новый параметр)
- `at` - скорректировать время для всех сегментов (новый параметр)
- `ct` - очистить устаревшие теги (новый параметр)

### Segment endpoint: `/segment`

- `s` - идентификатор потока (было `stream`)
- `f` - имя файла сегмента (было `file`)

## Примеры использования

### Получение манифеста с удалением последнего сегмента
```
GET /manifest?s=stream1&rl=1
```

### Получение манифеста с корректировкой времени
```
GET /manifest?s=stream1&at=1
```

### Получение манифеста с очисткой устаревших тегов
```
GET /manifest?s=stream1&ct=1
```

### Комбинированное использование
```
GET /manifest?s=stream1&dt=1&rl=1&at=1&ct=1
```

### Получение сегмента
```
GET /segment?s=stream1&f=22306abe87cb_main_seg195.ts
```

## Новые возможности

1. **Удаление последнего сегмента** (`rl=1`) - убирает последний сегмент из плейлиста
2. **Корректировка времени** (`at=1`) - корректирует время всех сегментов на основе их длительностей
3. **Очистка тегов** (`ct=1`) - удаляет устаревшие теги типа `#EXT-X-ALLOW-CACHE`
4. **Сокращенные параметры** - все параметры теперь короче для удобства
5. **Исправление дублирования PDT** - автоматически устраняет дублирование `#EXT-X-PROGRAM-DATE-TIME`
6. **Корректировка длительностей** - автоматически подстраивает `#EXT-X-TARGETDURATION`

## Формат URL сегментов

Теперь URL сегментов формируется с именем файла в конце:
```
/segment?s=stream1&f=filename.ts
```

Вместо старого формата:
```
/segment?stream=stream1&file=filename.ts
```

## Решение проблем HLS

### 1. Дублирование PROGRAM-DATE-TIME
Автоматически устраняется дублирование `#EXT-X-PROGRAM-DATE-TIME` между сегментами.

### 2. Несоответствие длительностей
Автоматически корректируется `#EXT-X-TARGETDURATION` на основе реальных длительностей сегментов.

### 3. Корректное время сегментов
Время каждого сегмента корректно рассчитывается на основе длительности предыдущего сегмента.

### 4. Очистка устаревших тегов
Удаляются устаревшие теги типа `#EXT-X-ALLOW-CACHE:NO`.
