# CHANGELOG - HLX Server

## Версия 2.1.0 (2025-08-21)

### Новые возможности
- ✅ **Сокращенные параметры** - все параметры теперь короче для удобства
- ✅ **Удаление последнего сегмента** - новый параметр `rl=1`
- ✅ **Корректировка времени** - новый параметр `at=1` для корректного расчета времени всех сегментов
- ✅ **Очистка тегов** - новый параметр `ct=1` для удаления устаревших тегов
- ✅ **Новый формат URL сегментов** - имя файла теперь в конце URL
- ✅ **Исправление дублирования PDT** - автоматическое устранение дублирования `#EXT-X-PROGRAM-DATE-TIME`
- ✅ **Корректировка длительностей** - автоматическая подстройка `#EXT-X-TARGETDURATION`

### Изменения в параметрах

#### Manifest endpoint (`/manifest`)
| Старый параметр | Новый параметр | Описание |
|----------------|----------------|----------|
| `stream` | `s` | Идентификатор потока |
| `delete_datetime`, `strip_pdt` | `dt` | Удалить дату/время |
| - | `rl` | Удалить последний сегмент |
| - | `at` | Скорректировать время для всех сегментов |
| - | `ct` | Очистить устаревшие теги |

#### Segment endpoint (`/segment`)
| Старый параметр | Новый параметр | Описание |
|----------------|----------------|----------|
| `stream` | `s` | Идентификатор потока |
| `file` | `f` | Имя файла сегмента |

### Примеры использования

#### Старый формат
```
GET /manifest?stream=stream1&delete_datetime=1
GET /segment?stream=stream1&file=segment.ts
```

#### Новый формат
```
GET /manifest?s=stream1&dt=1
GET /segment?s=stream1&f=segment.ts
```

#### Новые параметры
```
GET /manifest?s=stream1&rl=1          # Удалить последний сегмент
GET /manifest?s=stream1&at=1          # Скорректировать время
GET /manifest?s=stream1&ct=1          # Очистить теги
GET /manifest?s=stream1&dt=1&rl=1&at=1&ct=1 # Комбинированно
```

### Решение проблем HLS

#### 1. Дублирование PROGRAM-DATE-TIME
- Автоматически устраняется дублирование `#EXT-X-PROGRAM-DATE-TIME` между сегментами
- Каждый сегмент получает корректное время на основе длительности предыдущего

#### 2. Несоответствие длительностей
- Автоматически корректируется `#EXT-X-TARGETDURATION` на основе реальных длительностей сегментов
- Устраняется "стенка времени" - PDT теперь соответствует медиа-таймлайну

#### 3. Корректное время сегментов
- Время каждого сегмента корректно рассчитывается: `текущее_время + длительность_сегмента`
- Поддерживаются различные форматы времени (Z, +05:00, -07:00)

#### 4. Очистка устаревших тегов
- Удаляются устаревшие теги типа `#EXT-X-ALLOW-CACHE:NO`
- Улучшается совместимость с современными HLS плеерами

### Обратная совместимость
⚠️ **ВНИМАНИЕ**: Старые параметры больше не поддерживаются. Необходимо обновить все клиентские приложения для использования новых сокращенных параметров.

### Технические детали
- Добавлена функция `extractDuration()` для извлечения длительности сегментов
- Добавлена функция `adjustSegmentTiming()` для корректного расчета времени сегментов
- Добавлена функция `parsePDTTime()` для парсинга различных форматов времени
- Добавлена функция `adjustSegmentDurations()` для корректировки целевой длительности
- Переработана функция `transformManifest()` для поддержки новых возможностей
- Улучшена обработка сегментов, их длительностей и времени
- Добавлена поддержка очистки устаревших тегов
