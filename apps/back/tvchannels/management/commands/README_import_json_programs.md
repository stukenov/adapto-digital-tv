# Импорт программ из JSON файлов

## Описание

Django management команда `import_json_programs` предназначена для импорта телевизионных программ из JSON файлов, расположенных на удаленном сервере, через SSH соединение.

## Возможности

- SSH подключение к удаленному серверу
- Поиск JSON файлов в указанной папке (с опцией рекурсивного поиска)
- Парсинг JSON данных с настраиваемыми полями
- Обработка времени с учетом часового пояса Aktau (Asia/Aqtau)
- Автоматическое вычисление времени окончания программ
- Пакетный импорт с ограничением количества записей
- Проверочный режим (dry-run) без сохранения в базу данных

## Параметры команды

### SSH параметры (обязательные)
- `--ssh-host` - SSH хост для подключения
- `--ssh-user` - SSH пользователь
- `--ssh-password` - SSH пароль
- `--ssh-port` - SSH порт (по умолчанию: 22)

### Параметры файлов
- `--remote-path` - Путь к файлу или папке на удаленном сервере (обязательный)
- `--recursive` - Рекурсивный поиск JSON файлов в подпапках

### Параметры канала (один из параметров обязательный)
- `--channel-id` - ID телеканала
- `--channel-slug` - Slug телеканала

### Параметры JSON полей
- `--title-field` - Поле для названия программы (по умолчанию: "title")
- `--start-time-field` - Поле для времени начала (по умолчанию: "start_minute_pretty")
- `--programs-array-field` - Поле массива программ в JSON (по умолчанию: "program")

### Параметры импорта
- `--batch-size` - Количество записей для импорта за раз (по умолчанию: 100)
- `--max-records` - Максимальное количество записей для импорта
- `--dry-run` - Проверочный запуск без сохранения в БД

## Примеры использования

### Базовый импорт
```bash
python manage.py import_json_programs \
    --ssh-host 192.168.1.100 \
    --ssh-user admin \
    --ssh-password secret123 \
    --remote-path /data/tv_programs/ \
    --channel-id 1
```

### Импорт с рекурсивным поиском
```bash
python manage.py import_json_programs \
    --ssh-host tv-server.example.com \
    --ssh-user tvadmin \
    --ssh-password mypassword \
    --ssh-port 2222 \
    --remote-path /var/tv-data/ \
    --recursive \
    --channel-slug "first-channel" \
    --batch-size 50 \
    --max-records 1000
```

### Проверочный запуск
```bash
python manage.py import_json_programs \
    --ssh-host 10.0.0.50 \
    --ssh-user operator \
    --ssh-password pass123 \
    --remote-path /programs/today.json \
    --channel-id 3 \
    --dry-run
```

### Импорт с кастомными полями JSON
```bash
python manage.py import_json_programs \
    --ssh-host server.tv \
    --ssh-user user \
    --ssh-password password \
    --remote-path /data/ \
    --channel-slug "news-channel" \
    --title-field "show.name" \
    --start-time-field "schedule.start_time" \
    --recursive
```

## Формат JSON файлов

Команда поддерживает несколько форматов JSON файлов:

### Основной формат (ffplayout schedule)
```json
{
  "channel": "Channel Name",
  "date": "2025-01-20",
  "program": [
    {
      "duration": 2476.053333,
      "in": 0,
      "out": 2476.053333,
      "source": "/path/to/video.mp4",
      "start_minute_pretty": "2025-01-20 06:00",
      "title": "Мергендер мекені - 8 бағдарлама"
    },
    {
      "duration": 2508.8,
      "in": 2476.053333,
      "out": 4984.853333,
      "source": "/path/to/video2.mkv",
      "start_minute_pretty": "2025-01-20 06:40",
      "title": "Өнерлі өлке Нысана"
    }
  ]
}
```

### Массив расписаний (несколько дней)
```json
[
  {
    "channel": "Qazaq TV",
    "date": "2025-01-20",
    "program": [
      {
        "start_minute_pretty": "2025-01-20 06:00",
        "title": "Утренние новости"
      },
      {
        "start_minute_pretty": "2025-01-20 06:30",
        "title": "Погода"
      }
    ]
  },
  {
    "channel": "Qazaq TV", 
    "date": "2025-01-21",
    "program": [
      {
        "start_minute_pretty": "2025-01-21 06:00",
        "title": "Утренние новости"
      }
    ]
  }
]
```

### Кастомная структура (для старых форматов)
```json
{
  "show": {
    "name": "Документальный фильм",
    "description": "Интересный фильм о природе"
  },
  "schedule": {
    "start_time": "2025-01-20 14:00"
  }
}
```

**Примечание:** Поля `channel`, `date`, `duration`, `in`, `out`, `source` игнорируются при импорте. Используются только `title` и `start_minute_pretty` из каждого элемента массива `program`.

## Обработка времени

- Формат времени: `YYYY-MM-DD HH:MM` (например, "2025-08-11 06:00")
- Часовой пояс: Asia/Aqtau (Актау)
- Время окончания вычисляется автоматически как время начала следующей программы
- Для последней программы в списке добавляется 1 час по умолчанию

## Особенности

1. **Дубликаты**: Программы с одинаковым временем начала на одном канале пропускаются
2. **Сортировка**: Программы автоматически сортируются по времени начала перед импортом
3. **Валидация**: Проверяется корректность временных интервалов
4. **Безопасность**: Поддержка проверочного режима для тестирования
5. **Производительность**: Пакетное создание записей в базе данных

## Логирование

Команда выводит подробную информацию о процессе импорта:
- Статус SSH подключения
- Количество найденных файлов
- Прогресс обработки
- Ошибки парсинга
- Количество импортированных записей

## Требования

- Python пакеты: `paramiko`, `pytz`
- SSH доступ к удаленному серверу
- Существующий телеканал в базе данных
